# خريطة الميزات (Feature Map)

> هذه الوثيقة تربط كل ميزة رئيسية في النظام بـ: مكوّن الواجهة (UI Component)، أمر Tauri (Tauri Command)، الدالة في Rust (Rust Function)، الجداول المتأثرة (SQL Tables)، الاختبارات (Tests)، والثوابت المحاسبية المرتبطة (Invariants). الهدف هو تمكين أي مهندس من تتبّع مسار ميزة كاملة من زر الواجهة إلى قاعدة البيانات ومروراً بكل قاعدة محاسبية تنطبق عليها.

## ملاحظات عامة

- كل أمر كتابة يفتح `db.transaction()` ويستدعي `require_admin_session(&db, None)` أو `require_admin_session(&db, session_token.as_deref())` ثم `db.commit()` أو `ROLLBACK` عند الخطأ. هذا يضمن atomicity كاملة.
- أوامر القراءة (مثل `get_cars`, `get_partner_transactions`, `get_agencies`, `get_financial_summary`) تستخدم `state.db.lock()` بدون transaction لأنها قراءة فقط (§1.3).
- الثوابت المحاسبية الإلزامية الـ20 موثقة بالتفصيل في `docs/ACCOUNTING_INVARIANTS.md`.
- فهرس الأوامر الحساسة مع تفاصيل الـ idempotency وtransaction boundary موجود في `docs/COMMAND_CATALOG.md`.
- مخطط قاعدة البيانات الكامل موثق في `docs/SCHEMA.md`, وسجل الترحيلات في `docs/MIGRATIONS.md`.
