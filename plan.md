# خطة التسليم النهائية

الحالة: **GO**

## النطاق المعتمد

- صحة الحسابات والأموال ومنع الاحتساب المزدوج.
- ربط القيود والحركات بالمعرّفات الرقمية لا بالأسماء أو النصوص.
- السماح بدورات سيارات مستقلة تحمل رقم الشاصي نفسه دون اختلاط بياناتها.
- منع إنشاء الوكالة نفسها مرتين عند النقر المزدوج أو إعادة إرسال الطلب.
- اعتماد الدخول الافتراضي `admin/admin` دون تغيير إجباري.
- عدم تضمين النسخ الاحتياطي أو الاستعادة أو منظومات الاختبار والتقارير الثقيلة.

## ما تم

- schema v49 يضيف `source_entity_id` و`related_entity_id` و`reference_entity_id`، ويرحّل الروابط القديمة بصورة fail-closed.
- أزيلت هويات الأقساط المركبة، وأصبحت مرتبطة بـ`installments.id`.
- تحولت مطابقة مسارات الكتابة المحاسبية إلى الحقول الرقمية.
- بقي رقم الشاصي غير فريد عمدًا، وكل دورة شراء تعتمد على `cars.id` المستقل.
- أصبحت حماية تكرار الوكالة تعتمد على معاملة واحدة ورمز إنشاء وبصمة طلب حديثة بدقة أجزاء الثانية.
- أزيلت أوامر وشاشة وخيط النسخ الاحتياطي والاستعادة.
- أزيلت تقارير ومنظومات E2E/Oracle/randomized والسكربتات المتكررة، وأبقيت فحوص التسليم القصيرة فقط.

## بوابة GO المختصرة

- `cargo check --lib`
- `python3 scripts/check_numeric_write_identity.py`
- `python3 scripts/check_exact_money_sql.py`
- اختبار Rust الموجّه لتكرار الشاصي واستقلال `car_id`.
- اختبار Rust الموجّه لمنع تكرار رمز إنشاء الوكالة.
- اختبارات الواجهة المختصرة للأموال والقواعد المحاسبية ومنع التكرار والسيارات.
- TypeScript وبناء الواجهة.

أي ميزة جديدة خارج هذا النطاق تعامل كتطوير لاحق، ولا تعيد المشروع إلى دوامة بوابات تسليم غير مرتبطة بعمله المحاسبي الأساسي.
